# steam\_status\_keeper.vbs

《蒸汽平台\_状态\_保持器.可视化基本脚本（VBS）说明书》

一个VBScript脚本，用于非Steam游戏，适用于中文环境。

脚本主要是为了解决非Steam游戏状态保持的问题。当：

1. 游戏入口程序是一个模拟器，并且能启动多个游戏时；

2. 游戏入口程序事实上只是一个启动器，并在游戏运行后自动退出时；

3. 游戏入口程序本身是其他游戏的一部分，需要其他游戏启动时；

4. 单纯只是想维持一个状态时，

这个脚本可以保持Steam游戏状态。

### 举几个例子：

1. MuMu模拟器，本质上是为Android游戏模拟了一个Android虚拟机环境。这个环境只需要启动一次，但游戏可不是。如果在模拟器里启动了多个游戏，或虚拟机多开，那么Steam最多只能显示其中一个游戏的状态——并且如果在Steam中手误点了“停止”，整个模拟器都会停止运行！

2. 米哈游启动器（HoYoPlay CN），此类启动器多使用Web技术框架（尚不确定它是不是Electron，不过Web是没跑了），而基于诸如Electron之类框架的应用一般会集成版本管理。问题就出在版本管理。这会导致启动器安装根目录下的程序也只是一个启动器（没错，启动器的启动器），而这个启动器的启动器会在启动启动器后自动退出。这样，Steam就没法保持游戏状态，因为它直接启动的是启动器的启动器，它不会管启动器的启动器启动了什么，它只看到启动器的启动器退出了，就认为启动器自己结束了。
   有点绕。这里套了三层：第一层是米哈游启动器入口程序，检查完环境，启动第二层，完事就退出了，平时我们启动的就是它；第二层是米哈游启动器本体，被第一层启动；第三层是游戏本体，被第二层启动。

3. 原神·千星沙箱，米哈游会告诉你它只能在原神里启动，事实上也如此。不过这个严格来说不算“游戏”，因为它是游戏编辑器。

4. 状态随意咯。

## 使用方法

1. 把脚本放到任意位置，为了方便可以放在游戏根目录；

2. 在Steam中添加脚本为非Steam游戏；

3. （只想维持状态就跳过这一步）
   
   在启动选项处填写游戏入口程序目录（最好带引号），后附原本的启动参数（若有）；

4. 美美地开始游戏。

如想要结束游戏状态，在Steam中点击“停止”。状态不会自动结束。
